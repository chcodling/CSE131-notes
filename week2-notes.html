  Instructor Explanations

Instructor Explanation:

  • Briefly explain the purpose of JavaScript (client-side scripting language for web development).
  • Explain the importance of version control, especially Git, and how GitHub is used to store and manage code online.

 

  Level 1: Instructional Mastery

Activity Description:
Students will follow detailed instructions to:

  • Create a basic HTML file with embedded JavaScript to display a message on a webpage.
  • Learn basic Git commands to initialize a repository and push code to GitHub.

HTML Setup with JavaScript:

  • Open a text editor (VS Code or similar).
  • Create a basic index.html file with the following code:
    • Save and open the file in a browser to see the "Welcome to JavaScript!" alert.
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>JS Basics</title>
</head>
<body>
    <h1>Hello World</h1>
    <script>
        // Your first JavaScript code
        alert("Welcome to JavaScript!");
    </script>
</body>
</html>

Git and GitHub Setup:

  • Git Init: In the terminal, navigate to the folder containing index.html and initialize a git repository: 
    git init
  • Add and Commit Files: 
    git add .
    git commit -m 'Initial commit - added index.html with basic JS'
  • GitHub Repo Setup:
    Walk students through creating a GitHub account (if needed) and creating a new repository on GitHub.
  • Push to GitHub: 
    git remote add origin <your-github-repo-url> 
    git branch -M main 
    git push -u origin main

Tools & Resources:

  • Basic text editor (e.g., VS Code)
  • A modern browser (e.g., Chrome, Firefox)
  • Git installed on local machines
  • GitHub account

 

  Level 2: Research and Reporting

  • Activity Description:
    Students will conduct independent research on the following topics:

    • What is the purpose of JavaScript in modern web development?
    • What are three common Git commands, and what do they do?
    • What are the key differences between Git and GitHub?

  • Reporting Format:

    • Students will write a short summary (3-4 sentences per question) and submit it via a shared document or repository.

  • Tools & Resources:

    • Google search
    • Git and GitHub documentation (https://git-scm.com/doc and https://docs.github.com)

  Level 3: Analytical Recommendation

  • Activity Description:
    Students will evaluate different ways to link JavaScript to HTML and recommend the best approach. They can choose between inline JavaScript (within <script> tags in HTML), external JavaScript files, or other methods they find.

  • Demonstration:
    Students will present their recommendations with an explanation of pros and cons. For example:

    • Inline JavaScript: Easy to write for small scripts but hard to maintain for larger codebases.
    • External JavaScript: Better organization and reusability but requires additional file management.

  • Tools & Resources:

    • Web tutorials on linking JavaScript to HTML
    • Classroom discussion or Google Docs for presenting their findings

 

  Level 4: Decision Implementation

Activity Description:
Each student will decide on a method for linking JavaScript to their HTML file based on the analysis in Level 3. They will implement it by creating an external .js file, modifying their index.html to link to this file, and pushing the changes to GitHub.

Step-by-Step:

  • Create a new file called script.js.
  • Move the JavaScript code from the HTML file into script.js: 
    // script.js 
    alert("Welcome to JavaScript!");
  • Modify the index.html to link the external JavaScript file: 
    <script src="script.js"></script>
  • Use Git commands to add, commit, and push these changes to their GitHub repository: 
    git add .
    git commit -m "Moved JS to external file"
    git push

  • Tools & Resources:

    • Text editor (VS Code)
    • GitHub repository

 

  Conclusion & Reflection

Class Discussion:

  • Ask students to reflect on the process of creating JavaScript files and using GitHub.
  • Encourage them to ask any questions they may have on Git, JavaScript, or GitHub.

 

 Assessment of Each Level

  • Level 1: Instructional Mastery

    • Assessment:
      Students are assessed on their ability to correctly follow instructions to build a basic HTML file with JavaScript and initialize a Git repository.
    • Outcome:
      Successful display of "Hello World" and "Welcome to JavaScript!" message in the browser, and code pushed to a GitHub repository.

  • Level 2: Research and Reporting

    • Assessment:
      Students are assessed on the quality and clarity of their responses to the research questions. The focus is on their understanding of JavaScript's role, basic Git commands, and the differences between Git and GitHub.
    • Outcome:
      Clear, concise responses demonstrating a basic understanding of JavaScript and Git.

  • Level 3: Analytical Recommendation

    • Assessment:
      Students are assessed based on the depth of their analysis and reasoning behind their chosen method for linking JavaScript to HTML.
    • Outcome:
      Well-supported recommendations with explanations of why their chosen approach is most effective for their project.

  • Level 4: Decision Implementation

    • Assessment:
      Students are assessed on their ability to independently implement their chosen solution and successfully update their GitHub repository.
    • Outcome:
      Proper use of an external JavaScript file, modifications successfully pushed to the GitHub repository.
